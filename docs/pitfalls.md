# 运行时陷阱

本文记录代码 review、类型检查或语法检查不一定能发现的陷阱。

## 状态标签

| 标签 | 含义 |
|---|---|
| `[已解决]` | 项目代码或检查已经能防止该问题。 |
| `[已缓解]` | 有运行时保护，但仍需要设计纪律。 |
| `[框架行为]` | 根因在框架行为，只能调整项目设计。 |
| `[API 限制]` | 根因在外部 API 能力边界，只能通过工程约束规避。 |

## 1. Import 阶段 vs Runtime 阶段

问题：

- [框架] 会反复 import 入口文件。
- 模块级外部 I/O 可能在 runtime config 准备好之前失败。

规则：

- 外部 API 调用、元数据查询和昂贵计算必须放进 runtime function、task、handler 或 service。

## 2. 字符串契约

问题：

- Artifact key、route key、asset URN、function name、resource id、table name 经常没有编译器保护。

规则：

- 定义共享常量或 registry。
- 如果字符串被多个组件消费，添加契约检查。

## 3. 本地存储假设

问题：

- 本地文件在单进程开发环境可用，但跨 worker、pod 或机器会失败。

规则：

- 文档化 artifact storage 语义。
- 跨 worker artifact 使用共享对象存储。
- 关键场景添加跨运行单元写读的 smoke test。

## 4. 外部 API 状态

问题：

- SaaS API 常常没有事务、乐观锁或一致列表。

规则：

- 对同一外部资源串行写入。
- 关键写入后读回校验。
- 支持时使用 idempotency key 或稳定资源名。

## 5. 派生产物

问题：

- 唯一真源规则变化后，生成文件容易过期。

规则：

- 文档必须写明生成命令。
- CI 应在派生产物过期时失败。

## 6. 部署漂移

问题：

- 本地、测试、生产路径悄悄分叉。

规则：

- 只保留一个生产真源。
- 当目标是部署验收时，本地拓扑预览应 render 或执行生产路径。
