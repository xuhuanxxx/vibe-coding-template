# Review Checklist

用于 code review、自查和完成前检查。

## 正确性

- [ ] 改动发生在所属边界，而不是只修表层症状。
- [ ] 唯一真源仍然唯一。
- [ ] 派生产物已重新生成。
- [ ] 强关系没有藏进通用 JSON 或弱关系。
- [ ] 错误路径保留了可诊断上下文。

## 架构

- [ ] 依赖方向仍符合 `docs/architecture.md`。
- [ ] 入口文件只做编排，不承载可复用业务逻辑。
- [ ] 通用工具没有混入业务语义。
- [ ] 外部 SDK 行为被封装且可测试。

## 契约

- [ ] API/data/UI/deployment 契约随行为变化同步更新。
- [ ] 稳定 selector 或 resource name 未被意外破坏。
- [ ] 权限、事务、幂等和审计边界仍被保留。
- [ ] 当前文档描述当前事实，而不是未来计划。

## 验证

- [ ] 聚焦测试或 smoke 覆盖了被改行为。
- [ ] 本轮实际运行了相关 targeted gate。
- [ ] 已运行默认项目门禁，或明确说明跳过原因。
- [ ] 无法执行的外部验证已列为剩余风险。

## Git

- [ ] 未回退用户无关改动。
- [ ] 大改动已拆成语义化 commit。
- [ ] history rewrite 已创建 backup 分支。
- [ ] commit message 说明意图和契约影响。
